Hyperparameters, Regularization, Cost Functions, and Optimization

This master table illustrates how the Model, Cost Function, and Optimization Algorithm intertwine with specific hyperparameters and regularization methods across Machine Learning and Deep Learning.

Category / Algorithm Model Cost Function (Loss) Optimization Algorithm Hyperparameter Description Typical Range / Considerations Regularization Methods
General ML/DL N/A N/A N/A Learning Rate ($\eta$) Controls step size at each iteration. Small (0.001, 0.01) for precision; Large (0.1, 0.5) for speed but risks overshooting. L1, L2, Elastic Net, Early Stopping
Batch Size Samples processed before internal parameter updates. Small (1, 32) is noisy/frequent; Large (256, 1024) is smooth but risks poor generalization. N/A
Epochs Number of full passes through the training set. Too few = underfitting; Too many = overfitting. Early Stopping
Optimizer Algorithm used to update weights and minimize loss. SGD, Adam, RMSprop, Adagrad, Adadelta. Each has specific default rates. N/A
Regularization Strength Controls penalty applied to model complexity. L1 (Lasso) and L2 (Ridge). Higher values increase penalty. L1, L2, Elastic Net
Linear Models Linear Regression Mean Squared Error (MSE) Gradient Descent, OLS Alpha ($\alpha$) Regularization parameter in Lasso/Ridge. Larger values imply stronger regularization. L1, L2, Elastic Net
Logistic Regression Binary Cross-Entropy (Log Loss) GD, L-BFGS, SAGA C Parameter Inverse of regularization strength. Smaller $C$ = Stronger regularization. L1, L2
Fit Intercept Whether to calculate an intercept term. True / False N/A
Solver Optimization algorithm choice. 'liblinear', 'saga', 'lbfgs' based on data size. N/A
Tree-based Models Decision Tree Gini, Entropy, MSE Greedy Algorithm Max Depth Maximum depth of the tree. Higher depth = risk of overfitting. Pruning
Min Samples Split Minimum samples to split an internal node. Higher values prevent over-specialization. Pruning
Min Samples Leaf Minimum samples required at a leaf node. Higher values lead to more generalized trees. Pruning
Criterion Function to measure split quality. 'gini', 'entropy' (class); 'mse' (reg). N/A
Random Forest Gini / Entropy Bagging (Ensemble) N Estimators Number of trees in the ensemble. More trees improve accuracy but increase cost. Bagging, Feature Subsampling
Gradient Boosting MSE, Log Loss Additive Modeling Subsample Fraction of samples used for each base learner. Reduces variance/overfitting. Subsampling, Shrinkage
SVMs SVC / SVR Hinge Loss, Epsilon-Insensitive Quadratic Programming Kernel Function to transform data to higher dimensions. 'linear', 'poly', 'rbf', 'sigmoid'. C Parameter, L2
C Parameter Penalty for misclassification (soft margin). Smaller $C$ = Smoother decision surface. Soft Margin Regularization
Gamma ($\gamma$) Kernel coefficient (influence radius). Low = large radius; High = complex boundary. Complexity Control
Neural Networks FNN / CNN / RNN MSE, Cross-Entropy SGD, Adam, RMSprop Hidden Layers / Neurons Architecture depth and capacity. More = higher capacity but risk of vanishing gradients. Dropout, L1/L2, Batch Norm
Activation Function Non-linear transformation at each neuron. ReLU, Sigmoid, Tanh, Softmax. N/A
Dropout Rate Probability of dropping neurons during training. Typically 0.2 to 0.5. Dropout
Recurrent Dropout Dropout applied to recurrent connections (RNN). Prevents overfitting in sequences. Recurrent Dropout

Note: Optimization in Deep Learning often requires balancing the Batch Size and Learning Rate simultaneously to ensure the model finds a stable global minimum.
