The Transformer Algorithm

A comprehensive architectural breakdown from historical lineage to token-by-token execution.

Evolutionary Lineage: From RNNs to Attention

The Transformer is a deep learning architecture introduced in the landmark 2017 paper "Attention Is All You Need" by Google researchers. Before its emergence, sequential natural language processing relied heavily on Recurrent Neural Networks (RNNs) and Long Short-Term Memory (LSTM) networks. These models processed tokens step-by-step, creating an architectural bottleneck:

The Recurrent Bottleneck (RNN/LSTM)

Processed words sequentially. They compressed the entire historical context vector into a single fixed-size bottleneck, making it slow to compute and struggling to preserve dependencies across long token spans.

The Transformer Parallel Breakthrough

Cuts the sequential chain using Self-Attention. It allows the network to ingest and analyze all tokens in a sequence simultaneously, modeling explicit pairwise relationships regardless of structural distance.

A significant intermediate leap occurred in 2015 when Bahdanau et al. introduced an attention mechanism for sequence-to-sequence tasks. Instead of forcing the decoder to rely on a single static compressed vector, it empowered the decoder to actively look back across all encoder hidden states at each generation step, calculating dynamic alignment scores.

However, this early attention was still bound to underlying sequential recurrent layers. The Transformer eliminated the recurrent layer entirely. For instance, in the sentence "The dog didn't cross the road because it was tired", the model handles long-range coreference directly, calculating relationship scores to map the pronoun "it" explicitly back to "dog".

The Core Idea in One Line

Instead of reading word by word → Look at ALL words at once and learn relationships between them.

Consider this text example parsed by the architecture:
"The cat sat on the mat because it was soft"

When processing the pronoun "it", the attention mechanism distributes dynamic scalar scores across the contextual tokens. These softmax-normalized weights are optimized during training:

  • it → cat : 0.05 (low contextual relationship)
  • it → mat : 0.85  ← Resolved Target (High semantic relationship because "it" refers to "mat")
  • it → sat : 0.10 (low contextual relationship)

Self-Attention Mathematical Derivation ($Q, K, V$)

The Query, Key, and Value Intuition:

Every token projection matrix queries the surrounding space by formulating three operational vector profiles:

  • $Q$ (Query): "What descriptive properties am I searching for?"
  • $K$ (Key): "What semantic identities and profiles do I contain?"
  • $V$ (Value): "What actual informational payload do I yield or pass forward?"

Step-by-Step Mathematical Flow

Step 1: Input Vector Matrix Construction ($X$)

Given a baseline sentence layout: "Cat sat mat", each word is mapped to a static embedding space ($d_{\text{model}} = 4$):

$$X = \begin{bmatrix} 0.2 & 0.8 & 0.1 & 0.5 \\ 0.9 & 0.1 & 0.7 & 0.3 \\ 0.4 & 0.6 & 0.2 & 0.8 \end{bmatrix} \begin{matrix} \leftarrow \text{Cat} \\ \leftarrow \text{sat} \\ \leftarrow \text{mat} \end{matrix}$$

Step 2: Linear Projection using Learned Weight Parameters

The shared input matrix $X$ undergoes projection via specialized learned transformation weights $W^Q, W^K, W^V$ to initialize the operational matrices:

$$Q = X \times W^Q \quad \lvert \quad K = X \times W^K \quad \lvert \quad V = X \times W^V$$

Step 3: Calculating Scaled Dot-Product Attention Scores

Pairwise token alignment is computed by multiplying the Queries matrix directly by the transposed Keys matrix ($Q \times K^T$). This is scaled by the square root of the key dimension ($\sqrt{d_k}$) to prevent extreme variance bounds in deep networks:

$$\text{Score Matrix} = \frac{Q \times K^T}{\sqrt{d_k}} = \begin{bmatrix} 1.0 & 0.2 & 0.1 \\ 0.3 & 1.0 & 0.2 \\ 0.1 & 0.2 & 1.0 \end{bmatrix}$$

Step 4: Softmax Normalization

We apply a row-wise Softmax operation to transform these raw scores into positive probability values that sum to 1.0:

$$\text{Softmax}\left(\frac{Q \times K^T}{\sqrt{d_k}}\right) = \begin{bmatrix} 0.70 & 0.20 & 0.10 \\ 0.25 & 0.65 & 0.10 \\ 0.10 & 0.15 & 0.75 \end{bmatrix} \begin{matrix} \leftarrow \text{Cat attends 70\% to itself, 20\% to sat, 10\% to mat} \\ \leftarrow \text{sat attends 25\% to Cat, 65\% to itself, 10\% to mat} \\ \leftarrow \text{mat attends 10\% to Cat, 15\% to sat, 75\% to itself} \end{matrix}$$

Step 5: Aggregating the Final Value Vectors Matrix

The normalized attention coefficient weights filter the Value matrix ($V$) via matrix multiplication, outputting a context-enriched token map:

$$\text{Output} = \text{Softmax}\left(\frac{Q \times K^T}{\sqrt{d_k}}\right) \times V$$

The Universal Formulation

$$\text{Attention}(Q,K,V) = \text{Softmax}\left( \frac{Q \times K^T}{\sqrt{d_k}} \right) \times V$$

Multi-Head Attention Optimization

Rather than tracking relationships through a single attention mechanism, the model parallelizes the operation across $H$ distinct "heads." Each head maintains a completely separate set of projection weights ($W^Q_i, W^K_i, W^V_i$). This lets the model focus on different aspects of the text simultaneously:

  • Head 1: Tracks core syntax and grammatical structures.
  • Head 2: Maps long-range semantic coreferences and pronouns.
  • Head 3: Evaluates immediate spatial or positional tokens.
$$\text{MultiHead}(Q, K, V) = \text{Concat}(\text{Head}_1, \text{Head}_2, \dots, \text{Head}_H) \times W^O$$

The Internal Anatomy of a Modern LLM (e.g., Claude)

Large Language Models stack dozens of these individual Transformer layers together (typically 96+ layers in production variants) to build deep, abstract reasoning capabilities.

How Tokens Move Through the Stack Together

A common misconception is that models process text word-by-word through the layers. In reality, all tokens in a sequence travel through all layers together at the exact same time as a unified data matrix.

Let's map out how the sequence "The cat sat on the mat" moves through the network step-by-step to predict the next word ("and"):

[Execution Map: Layer Stack Pass-Through]
Input Tokens: [T1: "The", T2: "cat", T3: "sat", T4: "on", T5: "the", T6: "mat"]
↓ All 6 tokens are converted to dense vectors and stacked into a matrix

Layer 1 Execution (All 6 Vectors Enter Simultaneously):
• Self-Attention: "mat" vector calculates dynamic relationships with "The", "cat", "sat", "on".
• Feed-Forward Network: Processes each vector column independently.
• Output: $T_1', T_2', T_3', T_4', T_5', T_6'$ (Vectors updated with basic grammar context)
↓ Layer 1 outputs feed directly as inputs into Layer 2
Layer 2 Execution (All 6 Refined Vectors Enter):
• Learns subject-action pairings (e.g., matching the action "sat" to the subject "cat").
• Output: $T_1'', T_2'', T_3'', T_4'', T_5'', T_6''$
↓ Sequential layer flow loops through N layers...
Final Layer N Execution:
• Builds a deep semantic map of the entire scene (a calm cat resting on a surface).
• Output: $T_1^N, T_2^N, T_3^N, T_4^N, T_5^N, T_6^N$ (Full context-rich matrix)
The Next-Token Prediction Filter:
To generate the next word, the model checks only the final representation of the last token ($T_6^N$ "mat"). Because of the self-attention mechanism, this vector has already ingested information from all preceding words.

Output Softmax Layer Probability Distribution:
• "and"    → 0.35 (Highest Probability Target → Selected)
• "then"   → 0.20
• "."      → 0.18

Once "and" is generated, it is appended to the input sequence. The new sequence, "The cat sat on the mat and" (now 7 tokens), is passed through all $N$ layers again to predict the next token: "slept". This autoregressive loop repeats until an end-of-sequence token is generated.

The Mechanics Inside a Single Transformer Layer

As vectors travel through the network, the raw mathematical outputs can easily degrade or grow too large across 96+ layers. To keep values stable, each layer runs internal normalization and residual tracking pipelines:

Detailed Matrix Pipeline Between and Inside Layers:
Let's look at the transformations a single vector (like "cat") undergoes within one layer:
  1. Self-Attention: Token vectors interact to capture context.
  2. Residual Connection (Skip Layer): The raw incoming input vector is added back directly to the attention output: $$\text{Output}_{\text{residual}} = X + \text{Attention}(X)$$ Why? This preserves early features and prevents structural information from getting lost in deep networks.
  3. Layer Normalization: Standardizes the values across the vector dimensions to maintain a stable mean of 0 and variance of 1. This prevents numbers from exploding or vanishing.
  4. Feed-Forward Network (FFN): Passes the normalized vector through dense neural network layers for non-linear feature processing.
  5. Second Residual & Layer Norm: The FFN input is added back to its output and normalized one more time before entering the next layer.

How Large Language Models are Trained

Building production models like Claude requires a three-stage training pipeline:

1. Unsupervised Pre-Training

The model ingests massive datasets (websites, books, code). By predicting the next token billions of times, it naturally learns grammar, facts, world logic, and foundational reasoning patterns.

2. Supervised Fine-Tuning (SFT)

The pre-trained model is fine-tuned on high-quality, curated dialog templates. This teaches the system how to format answers and behave like an interactive, helpful AI assistant.

3. RLHF & Constitutional AI

Uses Reinforcement Learning from Human Feedback along with explicit safety rules (Constitutional AI). This step aligns the model's outputs with core principles like being helpful, harmless, and honest.
