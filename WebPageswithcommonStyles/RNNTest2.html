1. Recurrent Neural Networks

A Recurrent Neural Network (RNN) is fundamentally a neural architecture equipped with memory, designed to process sequential data by looking at inputs one step at a time while retaining information from previous steps. You can think of it like reading a sentence: your understanding of the current word is entirely dependent on your memory of the words that came before it. In this tutorial, we only discuss about Language models using RNN. We shall not be discussing any other applications of RNN.

RNN for Next word generation Example Task: Predict the next word
Input sentence: "The cat sat on the ___"

Traditional neural networks, like the Multilayer Perceptron (MLP), process words in isolation because their neurons are only connected to the layers above or below them. Neurons within the same layer do not talk to each other. This lack of horizontal communication means they have no way to pass information from one time step to the next. Consequently, they lack the 'memory' required to see how the meaning of a word is shaped by the words that came before it. An RNN, however, reads "The" to initialize its memory, updates that memory with "cat" to recognize the subject, and continues to build context through "sat" and "on". By the time it reaches the final "the", its internal state has captured the entire narrative, enabling it to logically predict "mat".

Figure 1: An unfolded RNN processing the sequence "The cat sat on the". The nodes \(h_1\) through \(h_5\) represent the evolving hidden states updated by matrices U and W to predict "mat".

The Hidden State: How It Works

This mechanism is driven by the hidden state \(h_t\), which serves as the network's persistent memory. At each discrete time step, the RNN updates this state by mathematically combining the new input \(x_t\) with the preceding hidden state \(h_{t-1}\) using a learned weight matrix:

$$h_t = \tanh(Wh_{t-1} + Ux_t + b_h)$$
First-Principles Architecture: Words as Vectors
  • Vector Inputs: The network does not process text strings directly. Each word is converted into a continuous numerical word vector (embedding). The length of this word vector must exactly match the number of physical input neurons in the RNN.
  • Time Over Space: Long sequence data does not require more input neurons. The RNN reuses the exact same set of input neurons at every clock tick, cycling through one word vector at a time.
  • Cumulative Memory: The hidden states do not store raw historical word vectors. Instead, each sequential state (\(h_{t-2}, h_{t-1}, h_t\)) dynamically blends the current input vector with past summaries. Consequently, each state holds a running calculation of all words processed up to that step, with the final hidden state \(h_t\) preserving the context of the entire sequence.

This ensures that the "story" of the sequence flows continuously from the first input to the final prediction. This "first-principles" logic is the necessary foundation for transitioning from basic neural nets into more complex sequential architectures.

2. Initialization: The Start of Time

The chain shown in RNN.svg starts with the input \(x_{t-1}\) feeding into hidden unit \(h_{t-1}\). However, the hidden unit \(h_{t-1}\) also has a horizontal incoming connection W. In a recursive loop, time step \(t=1\) must inherit a memory from time step \(t=0\).

We must initialize the very first hidden state. This is typically done with a vector of zeros, representing that the network starts with a clean slate before reading the first input. Let \(h^{(t-2)}\) be our zero initialization:

$$h^{(t-2)} = \vec{0}$$

3. Step-by-Step Forward Math

We will now perform forward propagation for the sequence \([t-1, t, t+1, t+2]\) visible in our diagram.

Time Step \(t-1\): The First Input

The hidden state \(h^{(t-1)}\) is calculated by combining the initial memory (\(h^{(t-2)}\) via weight \(W\)) and the new input (\(x^{(t-1)}\) via weight \(U\)), then applying the \(\tanh\) activation.

Hidden State Computation: $$Z_h^{(t-1)} = W h^{(t-2)} + U x^{(t-1)} + b_h$$ $$h^{(t-1)} = \tanh(Z_h^{(t-1)})$$

Time Step \(t\): Updating the Memory

This step is the most critical to understand recurrence. When computing \(h^{(t)}\), we are not using zero; we are using the "summary" of information just computed in the previous step (\(h^{(t-1)}\)) arriving via connection W.

Hidden State Update: $$h^{(t)} = \tanh(W h^{(t-1)} + U x^{(t)} + b_h)$$ Output Prediction: $$o^{(t)} = \sigma(V h^{(t)} + b_o)$$

Time Step \(t+1\): The Sequence Continues

The recurrence rule applies identically here. Forward propagation continues chronologically, reusing the exact same shared weight matrices \(U, V, W\).

$$h^{(t+1)} = \tanh(W h^{(t)} + U x^{(t+1)} + b_h)$$ $$o^{(t+1)} = \sigma(V h^{(t+1)} + b_o)$$

Time Step \(t+2\): The End of the Path

This is the final visible step in diagram. Notice that \(h^{(t+2)}\) has a horizontal outgoing connection W pointing to a dashed line, indicating this memory will persist into the next (unseen) input step.

$$h^{(t+2)} = \tanh(W h^{(t+1)} + U x^{(t+2)} + b_h)$$ $$o^{(t+2)} = \sigma(V h^{(t+2)} + b_o)$$

Generalizing Forward Propagation: The Recursive Loop

In a real implementation, we don't write four separate functions for \(t-1\) through \(t+2\). We write a generalized rule that applies to any time step \(t\) within a for loop that iterates over the entire input sequence length \(T\).

The Recursive Forward Rule:

The forward pass of a standard RNN is defined by these two simple equations applied over and over again from \(t=1 \dots T\):

$$\text{Memory Update:} \quad h^{(t)} = \tanh(W h^{(t-1)} + U x^{(t)} + b_h)$$ $$\text{Output Projection:} \quad o^{(t)} = \text{softmax}(V h^{(t)} + b_o)$$

Parameter Sharing: The Crucial Concept

Trainees may ask: "Why don't we have \(W_1\) connecting step \(1\to2\), and \(W_2\) connecting step \(2\to3\)?\"

The answer defines modern deep learning: Parameter Sharing. Reusing \(U, V,\) and \(W\) at every step means the RNN is learning a single set of logical rules for processing sequential data (e.g., "The word after 'is' is likely an adjective or noun"), and it can apply those rules to sequences of any length.

4. Backpropagation Through Time (BPTT)

To train our RNN, we need to find the optimal weights (\(U, V,\) and \(W\)) that minimize our prediction errors. Just like in a traditional neural network, we do this by calculating the gradient of a loss function with respect to our weights, and then updating them using Gradient Descent.

Because the exact same weight matrices (\(U, V,\) and \(W\)) are reused at every single word in the sequence, a mistake made at the very end is actually caused by those weights acting across the entire past. To fix this mistake, we must calculate how much those weights contributed to the error at every individual step. We do this by virtually stretching out our recursive loops into a straight chain of events and tracking the error gradients backward step-by-step into the past—a process called Backpropagation Through Time (BPTT).

Step 1: Calculating the Total Loss

For a single training sequence of length \(T\), we calculate an individual loss \(\ell(y^{(t)}, o^{(t)})\) at each time step by comparing our model's prediction \(o^{(t)}\) against the true target word \(y^{(t)}\). The total loss \(L\) for the entire sequence is simply the average of these individual losses across time:

$$L = \frac{1}{T} \sum_{t=1}^{T} \ell(y^{(t)}, o^{(t)})$$

Step 2: The Gradient Breakdown & The Chain Rule

To perform gradient descent, we must compute the partial derivatives of the total loss with respect to our three core weight matrices: \(\frac{\partial L}{\partial V}\), \(\frac{\partial L}{\partial U}\), and \(\frac{\partial L}{\partial W}\).

  • \(\frac{\partial L}{\partial V}\) (The Output Weights): This term is straightforward to calculate. Because the output matrix \(V\) connects directly to the local loss at each specific time step, its gradient can be calculated instantly for each isolated position without looking at the past.
  • \(\frac{\partial L}{\partial W}\) (The Recurrent Memory Weights): This is where standard backpropagation turns into Backpropagation Through Time. Because the hidden state \(h^{(t)}\) is calculated using the previous state \(h^{(t-1)}\), the derivative at any time step \(t\) must account for the entire history leading up to it.

Mathematical Underpinning of BPTT

By applying the calculus Chain Rule to the loss at any single time step \(t\), we can see how the recursive historical gradients are systematically unrolled through time.

Part 1: The Local Chain Rule Base $$\frac{\partial \ell(y^{(t)}, o^{(t)})}{\partial W} = \frac{\partial \ell(y^{(t)}, o^{(t)})}{\partial o^{(t)}} \cdot \frac{\partial o^{(t)}}{\partial h^{(t)}} \cdot \frac{\partial h^{(t)}}{\partial W}$$

The complexity lies in that last term, \(\frac{\partial h^{(t)}}{\partial W}\). Because \(h^{(t)}\) contains \(h^{(t-1)}\), the derivative expands recursively. For any given time step, the total influence of the weights on the current memory is the sum of its immediate impact plus its accumulated historical impact:

Part 2: The Temporal Multivariate Expansion $$\frac{\partial h^{(t)}}{\partial W} = \frac{\partial \tanh^{(t)}}{\partial W} + \frac{\partial \tanh^{(t)}}{\partial h^{(t-1)}} \cdot \frac{\partial h^{(t-1)}}{\partial W}$$

If we unroll this recursive equation all the way back to the first time step (\(t=1\")), we get the generalized BPTT equation. It uses a product series to multiply the gradients across every single step backward through time:

Part 3: Unrolling the Single-Step Recursion $$\frac{\partial h^{(t-1)}}{\partial W} = \frac{\partial \tanh^{(t-1)}}{\partial W} + \frac{\partial \tanh^{(t-1)}}{\partial h^{(t-2)}} \cdot \frac{\partial h^{(t-2)}}{\partial W}$$
Part 4: The Generalized Matrix Product Series (BPTT Equation) $$\frac{\partial h^{(t)}}{\partial W} = \sum_{i=1}^{t} \left( \prod_{j=i+1}^{t} \frac{\partial h^{(j)}}{\partial h^{(j-1)}} \right) \frac{\partial h^{(i)}}{\partial W}$$

Vanishing and Exploding Gradients in BPTT

BPTT suffers from the problem of Gradients either becoming very low values near to zero or very high values. Consider the equation \(\prod_{j=i+1}^{t} \frac{\partial h^{(j)}}{\partial h^{(j-1)}}\) identified in Part 4. To understand why this product behaves so volatilely over long historical sequences, we must compute the exact Jacobian matrix for a single time step. Given the recurrence relation \(h^{(j)} = \tanh(W h^{(j-1)} + U x^{(j)} + b_h)\), the partial derivative of the current hidden state with respect to the previous hidden state is given by:

$$\frac{\partial h^{(j)}}{\partial h^{(j-1)}} = \text{diag}\left(1 - \left(h^{(j)}\right)^2\right) \cdot W^T$$

When we substitute this expression back into the full product series across an extended temporal window stretching from step \(i\) to step \(t\), the overall gradient contribution expands to:

Part 5: Expanded Temporal Jacobian Product $$\prod_{j=i+1}^{t} \frac{\partial h^{(j)}}{\partial h^{(j-1)}} = \prod_{j=i+1}^{t} \left[ \text{diag}\left(1 - \left(h^{(j)}\right)^2\right) \cdot W^T \right]$$

By isolating this expression, we see that propagating an error backward across \((t - i)\) steps requires multiplying the recurrent weight matrix \(W^T\) by itself repeatedly. In linear algebra terms, the long-term behavior of this sequence depends entirely on the spectral radius (the largest absolute eigenvalue, \(\lambda_{\max}\)) of the weight matrix \(W^T\), coupled with the squashing property of the activation derivative:

  • The Vanishing Gradient Problem (\(\lambda_{\max} < 1\)): The derivative of the \(\tanh\) function, \((1 - (h^{(j)})^2)\), is strictly bounded between \(0\) and \(1\). If the maximum eigenvalue of your weight matrix \(W\) is also less than \(1\), the continuous matrix product acts as an exponential fraction multiplier. As the distance between the error step \(t\) and historical step \(i\) grows large, the product exponentially decays toward zero: $$\lim_{t - i \to \infty} \prod_{j=i+1}^{t} \frac{\partial h^{(j)}}{\partial h^{(j-1)}} = \mathbf{0}$$ Consequently, \(\frac{\partial L}{\partial W}\) receives absolutely zero meaningful feedback from early sequence inputs, rendering the model incapable of learning long-range dependencies.
  • The Exploding Gradient Problem (\(\lambda_{\max} > 1\)): Conversely, if the weights are initialized or updated such that the spectral radius of \(W\) is significantly greater than \(1\), and the hidden unit activations do not fully saturate the \(\tanh\) boundaries, the repeated multiplication of elements greater than \(1.0\) causes the gradient norm to scale exponentially: $$\| \text{Gradient} \| \propto (\lambda_{\max})^{t-i}$$ This exponential surge causes massive weight updates during gradient descent, which destabilizes training completely, forces optimization steps to oscillate wildly, and eventually triggers numerical overflow errors (\(\text{NaN}\)).

5. Algorithm: Backpropagation Through Time

Translating these high-level calculus rules into programmatic rule sets gives us the final training execution strategy. Notice how the backward pass steps backward through the sequence matrix (from time step \(T\) down to \(1\)) while building upon cumulative structural derivatives. 

Inputs: Sequences {x_1, ..., x_T}, Targets {y_1, ..., y_T}
Parameters: U, W, V, b_h, b_o

for i = 1 to epochs:
   for each sequence (x, y):
      
      # forward pass
      h_0 = 0
      for t = 1 to T:
         h_t = tanh(W * h_{t-1} + U * x_t + b_h)
         o_t = softmax(V * h_t + b_o)

      # backward pass
      dU, dW, dV, db_h, db_o = 0, 0, 0, 0, 0
      next_dh = 0

      for t = T down to 1:
         # current step error
         err_t = o_t - y_t
         
         # local output path (vertical)
         dV   += err_t * h_t^T
         db_o += err_t
         
         # future hidden state path (horizontal)
         # total derivative = local gradient + future gradient
         dh_total = (V^T * err_t) + (W^T * next_dh)
         
         # activation derivative (chain rule)
         dh_raw = dh_total * (1 - h_t^2)
         
         # accumulate structural gradients
         dU   += dh_raw * x_t^T
         dW   += dh_raw * h_{t-1}^T
         db_h += dh_raw
         
         # pass gradient to previous step
         next_dh = dh_raw

      # weight updates via gradient descent
      U   -= η * dU
      W   -= η * dW
      V   -= η * dV
      b_h -= η * db_h
      b_o -= η * db_o

6.1 From Raw Text to Training Pairs

Before we can run forward/backward pass, we need numbers, not words. This section converts sentences into the (input, target) tensors an RNN expects.

An RNN language model learns by predicting the next token given previous tokens. A token is usually a word, but can also be a subword or character. We build training examples by sliding a window.

One-Hot vs Embeddings: Why We Don't Use One-Hot

After mapping words to indices, the naive way to feed them to the RNN is one-hot encoding. For a vocabulary of size V=10,000, the word "cat" with index 245 becomes a vector of 9,999 zeros and a single 1 at position 245.

$$\text{onehot}("cat") = [0, 0, ..., 1, ..., 0] \quad \text{shape: } (V \times 1)$$

Two fatal problems:

  1. Size: With V=50k, each input is 50,000 neurons. Your U matrix becomes (H × 50000). GPU memory explodes.
  2. No meaning: "cat" and "dog" are equally distant from "pizza". One-hot encodes identity, not semantics.
Solution: Learned Embeddings
Instead, we use a small dense vector D=300. The embedding matrix E of shape (V × D) is a lookup table. Row 245 of E is the 300-dim vector for "cat". E is learned during training via BPTT, so "cat" and "dog" end up close together.
$$x_t = E[\text{idx("cat")}, :] \quad \text{shape: } (1 \times D) \text{ or } (D \times 1)$$

We treat it as a column vector (D × 1) to match U h_{t-1} + U x_t

7. Training Algorithm: Next-Word Generation from Scratch

This brings together the input vector lookup, forward recurrence loop, categorical cross-entropy evaluation, and Backpropagation Through Time (BPTT).

1. Phase 1: The Chronological Forward Sweep

The network cannot calculate any gradients until it knows how poorly it predicted the sequence.Forward Propogation needs to look at the full sentence first. Therefore, it completes the entire forward pass across all time steps before running backpropagation through time to update the weights."

  • The loop starts at t = 1 and runs sequentially up to t = T.
  • As it computes each step, the system caches (stores in RAM) the inputs xt, the hidden states ht, and the output probability distributions pt.
  • You must wait until time step T is reached to calculate the final total sequence loss (L).

2. Phase 2: The Reverse BPTT Sweep

Once the forward sweep is finished and the total loss is established, the backward pass begins at the final time step T and runs in reverse down to 1:

  • The error signal flows backward through the stored cache.
  • Because the hidden state at the last step (hT) was built on top of the hidden state before it (hT-1), the gradient must cascade through every single historical memory state sequentially.

The Engineering Bottleneck: Why this limits RNNs

Because you have to compute all forward steps before starting the backward pass, the computer must hold every single hidden state (h1, h2, ..., hT) inside its GPU/CPU memory simultaneously. If a text sequence is 10,000 words long, you have to store 10,000 dense matrix states in RAM before you can execute a single step of backpropagation. For long sequences, this quickly causes Out-of-Memory (OOM) errors.

To solve this practical limitation, engineers invented a modification called Truncated BPTT. Instead of processing the entire sequence at once, they split a long sequence into smaller chunks (e.g., 50 words at a time). The forward pass computes 50 steps, BPTT runs immediately back through those 50 steps, updates the weights, and then passes only the final hidden state value as the starting memory for the next 50-word chunk. 

Given Dimensions:
  V : Vocabulary Size
  D : Word Vector Dimension (Input Neurons)
  H : Hidden State Dimension (Memory Features)

Hyperparameters:
  η : Learning Rate (Scalar)
  MaxEpochs : Total number of full training cycles

Model Parameter Parameters:
  E   : Embedding Matrix of shape (V × D)
  U   : Input-to-Hidden Weight Matrix of shape (H × D)
  W   : Hidden-to-Hidden Weight Matrix of shape (H × H)
  V_w : Hidden-to-Output Weight Matrix of shape (V × H)
  b_h : Hidden Layer Bias Vector of shape (H × 1)
  b_o : Output Layer Bias Vector of shape (V × 1)

Algorithm Execution:

For Epoch = 1 To MaxEpochs:
    Initialize TotalLoss = 0
    Initialize Hidden State Vector Base: h_(-1) = ZeroVector(Shape: H × 1)
    
    # ------------------------------------------
    # 1. FORWARD PROPAGATION LOOP
    # ------------------------------------------
    For t = 1 To T:
        # Retrieve the specific word vector row from Embedding Matrix
        x_t = RowVector(E, InputWordIndex_t) transposed to shape (D × 1)
        
        # Calculate recurrence state memory
        Z_h = MatrixMultiply(W, h_(t-1)) + MatrixMultiply(U, x_t) + b_h
        h_t = tanh(Z_h)  # Shape: (H × 1)
        
        # Project memory to raw vocabulary layer scores
        Z_o = MatrixMultiply(V_w, h_t) + b_o  # Shape: (V × 1)
        
        # Softmax normalization to yield probability distribution Vector (p_t)
        For each vocabulary element k from 1 To V:
            p_t[k] = exp(Z_o[k]) / SumAcrossAllElements(exp(Z_o))
            
        # Accumulate categorical cross-entropy loss tracking metric
        TargetIndex = TargetWordIndex_t
        TotalLoss = TotalLoss - log(p_t[TargetIndex])
        
        Cache current states {x_t, h_t, p_t} for backward pass usage

    # ------------------------------------------
    # 2. BACKPROPAGATION THROUGH TIME (BPTT) LOOP
    # ------------------------------------------
    Initialize Local Gradient Accumulators to Zero:
      dU = MatrixZeros(H × D), dW = MatrixZeros(H × H), dV_w = MatrixZeros(V × H)
      db_h = VectorZeros(H × 1), db_o = VectorZeros(V × 1), dE = MatrixZeros(V × D)
    
    Initialize Future Step Temporal Error Carryover: next_dh = VectorZeros(H × 1)
    
    For t = T Down To 1:
        # Compute local categorical cross-entropy error vector gradient
        err_t = CopyVector(p_t)
        err_t[TargetWordIndex_t] = err_t[TargetWordIndex_t] - 1  # Subtract 1 from correct class target
        
        # Local vertical output weight updates gradient mapping
        dV_w = dV_w + MatrixTensorProduct(err_t, h_t.Transposed)
        db_o = db_o + err_t
        
        # Unroll historical horizontal hidden path memory tracking 
        dh_total = MatrixMultiply(V_w.Transposed, err_t) + MatrixMultiply(W.Transposed, next_dh)
        
        # Backpropagate through non-linear activation derivative layer
        dh_raw = dh_total ⊙ (ElementWiseOneVector - SquareElements(h_t))
        
        # Accumulate structural sequence gradients over time steps
        dU   = dU + MatrixTensorProduct(dh_raw, x_t.Transposed)
        dW   = dW + MatrixTensorProduct(dh_raw, h_(t-1).Transposed)
        db_h = db_h + dh_raw
        
        # Project error mapping backward to individual source input matrix rows
        dE[InputWordIndex_t] = dE[InputWordIndex_t] + VectorSqueeze(MatrixMultiply(U.Transposed, dh_raw))
        
        # Assign current raw hidden derivative to cascade backward into next historical time step
        next_dh = dh_raw

    # Apply Gradient Boundary Clipping constraints to arrest Exploding Gradients
    For each derivative matrix [dU, dW, dV_w, db_h, db_o, dE]:
        ClipAllElementsWithinBoundaries(Matrix, MinValue=-5.0, MaxValue=5.0)

    # ------------------------------------------
    # 3. CONVERGENCE PARAMETER OPTIMIZATION
    # ------------------------------------------
    U   = U   - (η × dU)
    W   = W   - (η × dW)
    V_w = V_w - (η × dV_w)
    b_h = b_h - (η × db_h)
    b_o = b_o - (η × db_o)
    E   = E   - (η × dE)
    
    Print Metrics: EvaluationSequenceLoss = TotalLoss / T

8. Execution: Evaluation and Inference Phase

Once training is complete, text generation is verified by providing a seed input vector token and calculating ongoing sequence steps recursively without target constraints: 

Inference Routine (SeedWordToken, NumberOfGenerationSteps):
    CurrentWordIndex = Map(SeedWordToken → Index)
    h = VectorZeros(Shape: H × 1)
    OutputSentenceSequence = [SeedWordToken]
    
    For Step = 1 To NumberOfGenerationSteps:
        # Vector Lookup Pass
        x = RowVector(E, CurrentWordIndex) transposed to shape (D × 1)
        
        # Propagate Recurrent Hidden Unit Sequence Memory
        h = tanh(MatrixMultiply(W, h) + MatrixMultiply(U, x) + b_h)
        
        # Calculate Output Probability Vector
        OutputVectorScores = MatrixMultiply(V_w, h) + b_o
        p = ComputeSoftmaxDistribution(OutputVectorScores)
        
        # Select target with maximal probability output matrix score
        CurrentWordIndex = FindIndexWithMaximalValue(p)
        PredictedStringToken = Map(CurrentWordIndex → Word)
        
        Append PredictedStringToken to OutputSentenceSequence
        
    Return OutputSentenceSequence

4. Long Short-Term Memory (LSTM)

The RNN Problem: Uncontrolled Memory
A standard RNN has only one memory vector, the hidden state h_t. At every step, it computes h_t = tanh(W h_{t-1} + U x_t). This forces it to completely rewrite its memory using both old memory and new input, even when the new input is irrelevant. It cannot choose to "keep" important old information or "ignore" noisy new input.

The LSTM Fix: Gated Memory
An LSTM adds a second memory called the Cell State C_t. Think of C_t as a conveyor belt running through time. Three gates control it:
  1. Forget Gate: Chooses what to erase from C_{t-1}
  2. Input Gate: Chooses what new info from x_t should be added to C_t
  3. Output Gate: Chooses what part of C_t to expose as h_t
Now the network can protect information for many steps, or dump it when it’s no longer needed. The gradient flows through C_t with fewer multiplications, so it vanishes slower. One LSTM cell at time t takes input x_t, previous hidden state h_{t-1}, previous cell state C_{t-1}, and produces new h_t, C_t. The 3 gates are just sigmoid layers that output numbers between 0 and 1.

LSTM Architecture

This is the standard LSTM block. The blue line is h^{(t-1)}, black line at top is cell state C^{(t-1)}, and black line at bottom is input x^{(t)}. Follow the arrows to see how information flows.

Walk through the diagram step by step:

Step 1: Gates look at h^{(t-1)} and x^{(t)}
The blue h^{(t-1)} and black x^{(t)} feed into all 4 yellow boxes. Each box is a neural layer with its own weights and bias.
GateEquationWhat it does in diagram
Forget f $$f = \sigma(W_{hf} h^{(t-1)} + W_{xf} x^{(t)} + b_f)$$ Left σ box. Output f goes up to "×" to scale old C^{(t-1)}
Input i $$i = \sigma(W_{hi} h^{(t-1)} + W_{xi} x^{(t)} + b_i)$$ Middle σ box. Output i goes to "×" before the "+"
Candidate Ĉ $$\tilde{C} = \tanh(W_{hc} h^{(t-1)} + W_{xc} x^{(t)} + b_c)$$ Right Tanh box. Creates new content, also goes to "×"
Output o $$o = \sigma(W_{ho} h^{(t-1)} + W_{xo} x^{(t)} + b_o)$$ Bottom σ box. Output o goes to final "×" on right
Step 2: Update the cell state C^{(t)} - the top line
C^{(t-1)} enters from left. First "×" multiplies it by f. This is the forget gate in action. Then "+" adds new info: i * Ĉ. Result is C^{(t)} going out right.
$$C^{(t)} = f * C^{(t-1)} + i * \tilde{C}$$
Step 3: Compute output h^{(t)} - the blue line
C^{(t)} goes through purple Tanh, then "×" with o. Result is h^{(t)}, which goes to next time step AND to next layer.
$$h^{(t)} = o * \tanh(C^{(t)})$$
BPTT from the Diagram
Look at the top black line C^{(t-1)} → C^{(t)}. There’s only one weight: multiplication by f. No repeated W matrices. If the network learns f ≈ 1, then ∂C^{(t)} / ∂C^{(t-1)} = f ≈ 1. Gradient flows backward through time with no decay. That’s why LSTM beats other RNNs on long sequences. The gates f, i, o still use W matrices, so they can have vanishing gradients. But C_t is protected.
  • Why 4 W matrices? Each gate needs to learn different behavior. f learns “when to forget”, i learns “when to write”. Same input, different weights.
  • What are ⊙ and ⊕? ⊙ = element-wise multiply, shown as "×" in diagram. ⊕ = element-wise add, shown as "+" in diagram.
  • Why tanh for Ĉ and C^{(t)}? Keeps cell values between -1 and 1, prevents explosion. Sigmoid for gates because we need values 0 to 1.

Alignment cases
