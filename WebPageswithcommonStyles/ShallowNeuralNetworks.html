Neural Networks

A Neural Network is a powerful Machine Learning model designed to identify complex patterns in data. It is the foundational technology behind modern AI, widely used for Pattern Recognition (such as identifying objects in photos), Image Generation, and Natural Language Processing (like predicting the next word in a sentence).

A Neural Network is one of the few Machine Learning models capable of handling massive amounts of high-dimensional data, such as millions of pixels in an image or billions of tokens in text, to train for accurate predictions. Compared to the early days of AI, modern Neural Networks are significantly larger, both in terms of learnable parameters (the weights) and the depth of internal processing (the number of hidden layers).

modern Neural Networks require massive computational infrastructure. Training these models on a standard CPU (Central Processing Unit) would take years because CPUs are designed for general-purpose tasks. Instead, modern AI relies on GPUs (Graphics Processing Units) and specialized chips like TPUs (Tensor Processing Units).These processors are built for massive parallelism, allowing them to perform thousands of mathematical calculations (like the matrix multiplications in our weighted sums) simultaneously. This hardware evolution, combined with large-scale data centers, is what allows us to train models with billions of parameters in a matter of weeks rather than decades.

From Linear Regression to Layers of Logistic Regression

Linear and Logistic Regression, as seen in earlier chapters, take inputs, multiply them by weights, and produce an output. A Neural Network is a "multi-layer" stack of linear and logistic regression units that introduces intermediate processing steps. This multi-layer structure allows the model to process data in stages.

In each layer, processing leads to the creation of new features. By using non-linear Activation Functions (like ReLU or Sigmoid) in hidden layers,

Math (Shallow NN): $$y = \sigma(W_2 \cdot \sigma(W_1x + b_1) + b_2)$$
the network automatically learns to combine raw inputs into complex features that best solve the task.

A neural networks broadly involves the following process
  1. (1) Forward Pass: Data is fed through the network to generate predictions.
  2. (2) Loss Calculation: A loss function measures how far off the predictions were from the true labels.
  3. (3) Backpropagation: The error is propagated backward through the network to compute the gradients (how much each weight should be adjusted).
  4. (4) Weight Update: The optimizer (e.g., Adam) uses the gradients to update the network's weights.
(1) - (4) will be repeated as desired no of loops

While the above steps are the same, the details of what the network is doing at each step are very different, Processing of NN say MLP layers are different from the processing of layers in CNN
  • 1. The Inputs (Layer L1): The Feature Vector The first layer, L1, is the starting point where your "raw features" live.
    • The "Real" Facts: Each neuron ($n_1, n_2, n_3$) represents a specific, measurable attribute. If you are identifying a "Cat," $n_1$ might be "Pointy Ears," $n_2$ might be "Whiskers," and $n_3$ might be "Tail Length."
    • No Abstraction Yet: These are the only values that are grounded in reality. In an MLP, we don't need the complex spatial filters used in images (CNNs); we just pass these direct facts to the first "team" of Logistic Regressors.
  • 2. The Weights ($W_{jk}^{(l)}$): Trust and Influence The lines are the Weights, representing the "influence" one neuron has on the next. Since each neuron in the next layer acts like its own LR model, these weights are essentially the coefficients ($\beta$ or $w$) of that model.
    • Positive Weight: A vote of confidence. If $n_1$ (Ears) is high, it increases the "probability" that the next neuron (e.g., "Cat-like shape") will fire.
    • Negative Weight: A "veto." If the input is $n_2$ (Barking), a negative weight tells the next neuron to not fire, because it's probably not a cat.
    • Zero Weight: The connection is irrelevant to that specific sub-decision.
  • 3. The Weighted Sum: The Linear Core Before the "Logistic" part happens, the neuron must perform a calculation. This is the linear regression step of the process. It gathers all incoming signals and accumulates them into a single value, $z$.

    The Accumulator: It calculates the weighted sum of all inputs plus a Bias ($b$), which acts like the intercept in a standard regression:

    $$z_j^{(l)} = \sum_{k} (w_{jk}^{(l-1)} \cdot n_k^{(l-1)}) + b_j^{(l)}$$
  • 4. The Activation Function: The "Logistic" Step This is where the neuron truly becomes a Logistic Regression unit. The weighted sum $z$ is passed through a function to decide its "signal strength."
    • The Probability Gate: If we use a Sigmoid activation, the neuron is exactly a Logistic Regression—it takes the sum and "squashes" it into a value between 0 and 1.
    • The Non-Linear Magic: While LR is appropriate here, we stack them because one "linear" decision isn't enough. By using functions like ReLU or Sigmoid, we allow the network to "bend" the rules, enabling it to learn relationships that aren't just straight lines.
  • 5. The Loss (The Final Verdict): The Teacher At the final layer (L4), the network gives its best guess. The Loss (or Cost) is the math that tells the network how wrong it was.
    • Measuring Error: If the final LR unit in the output layer predicts "90% Cat" but the ground truth is "Dog," the Loss function (like Cross-Entropy) generates a high penalty.
    • Optimization: The "Training" process uses this Loss to work backward. It says, "To make this Loss smaller next time, which weights ($W$) in our team of logistic regressors do we need to nudge up or down?"
  • The Calculus of ReLU: Why the Gradient is 1 It is important to distinguish between a constant and a linear slope. For all values where $z > 0$, ReLU acts as the identity function ($f(z) = z$).
    • Linear Slope: Because $f(z) = z$ in the positive region, its derivative is exactly 1. It is like a ramp with a 45-degree angle.
    • Not a Constant: If ReLU gave a constant value (like 5), the gradient would be 0 and the network would never learn. By providing a gradient of 1, it allows the "error signal" to pass through the neuron unchanged.
    • Dead Neurons: If $z < 0$, the gradient is 0. This is known as the "Dying ReLU" problem, where a neuron stops updating because it is stuck in the negative region.

    • Architectural Math: Parameter Counting

    To calculate the complexity of a Dense Layer, we multiply the input dimension by the output dimension.

    Input Size ($L_1$): n
    Output Size ($L_2$): n - 4
    --------------------------
    Connection Count: n * (n - 4) = n² - 4n

    Note: In Large Language Models, $n$ is often 4096 or higher. A single layer can have 16 million+ connections!
