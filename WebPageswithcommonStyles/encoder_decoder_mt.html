RNN Encoder–Decoder for Machine Translation

How an RNN reads a source sentence, compresses it into a context vector, and generates the translation word by word — including exactly what each decoding step relies on, and why the bottleneck matters.

1 The Task: Why Seq2Seq?

Machine translation was the original motivating task for the Encoder–Decoder framework. The goal is to take a sentence in one language — say English — and produce an equivalent sentence in another language, say French. The challenge is that input and output sentences can differ in length, word order, and grammatical structure, so a simple word-by-word mapping does not work.

The Encoder–Decoder architecture handles this by splitting the problem into two sequential stages — a reading stage (Encoder) and a writing stage (Decoder) — connected by a single compressed vector called the context vector c.

Example used throughout this document
Source (English): "The cat sat"
Target (French): "Le chat était"

We will trace exactly how each French word is generated, and what the Decoder relies on at every step.

2 The Encoder — Reading the Source

The Encoder is a standard RNN. It reads the English sentence one word at a time, updating its hidden state at each step. After the final word is processed, its last hidden state becomes the context vector c — a single fixed-size numerical vector that is supposed to carry the full meaning of the source sentence.

Figure 1: RNN Encoder reading "The cat sat". Hidden states h₁→h₂→h₃ carry memory forward. The final h₃ becomes the context vector c — the only information passed to the Decoder.

Encoder equations

$$h_t = \tanh(W h_{t-1} + U x_t + b_h) \quad \text{for } t = 1 \dots T$$ $$c = h_T \quad \text{(context vector = final hidden state)}$$

The same weight matrices W and U are reused at every step. At the end of the sequence, c = h₃ — a fixed-size vector containing the RNN's compressed understanding of the entire English sentence.

3 The Decoder — Generating the Translation

The Decoder is also an RNN, but its job is to generate rather than read. It starts with the context vector c as its initial hidden state and produces one French word at each time step.

Figure 2: RNN Decoder generating "Le chat était". The context vector c initialises the Decoder. At each step, the previously predicted word is fed back (dashed arrows) as the next input alongside the recurrent hidden state.

Decoder equations

$$s_0 = c \quad \text{(context vector initialises the Decoder)}$$ $$s_t = \tanh(W s_{t-1} + U y_{t-1} + b_h)$$ $$o_t = \text{softmax}(V s_t + b_o) \quad \Rightarrow \quad \hat{y}_t = \arg\max(o_t)$$

where y_{t-1} is the previously predicted word (fed back as input), and ŷ_t is the word with the highest probability at step t.

4 What Does Each Decoding Step Actually Rely On?

This is the most important question to understand deeply. Let us trace every dependency for generating "Le", "chat", and "était".

Step 1 Generating "Le" — relies on c and <s>

What s₁ depends on
  • Context vector c — arrives as s₀ = c, carrying the compressed meaning of the entire English sentence "The cat sat". This is the only direct link between the Encoder and the Decoder.
  • Start token <s> — since there is no previously predicted word yet, a special start token triggers the first prediction. It is converted to an embedding and fed in via matrix U.
  • Learned weight matrices W, U, V — do the actual computation.
$$s_1 = \tanh(\underbrace{W \cdot c}_{\text{context vector}} + \underbrace{U \cdot \text{emb}(\langle s \rangle)}_{\text{start token}} + b_h)$$ $$\text{"Le"} = \arg\max\,\text{softmax}(V \cdot s_1 + b_o)$$

The softmax produces a probability over the entire French vocabulary. "Le" wins because — given this context — it has the highest probability.

Step 2 Generating "chat" — c now indirect

What s₂ depends on — and this is where it gets critical
  • Previous hidden state s₁ via matrix W — s₁ was computed from c, so c is now one step removed.
  • Previously predicted word "Le" — fed back as input via matrix U.

Notice: c does not appear directly in this equation. It influences s₂ only through the chain s₁ → s₂.
$$s_2 = \tanh(\underbrace{W \cdot s_1}_{\text{c is one step removed}} + \underbrace{U \cdot \text{emb}(\text{"Le"})}_{\text{previous prediction}} + b_h)$$ $$\text{"chat"} = \arg\max\,\text{softmax}(V \cdot s_2 + b_o)$$

Step 3 Generating "était" — c two steps removed

$$s_3 = \tanh(W \cdot s_2 + U \cdot \text{emb}(\text{"chat"}) + b_h)$$ $$\text{"était"} = \arg\max\,\text{softmax}(V \cdot s_3 + b_o)$$

Now c has passed through two hidden state updates before influencing s₃. The signal from the original English sentence is becoming diluted.

5 The Dilution Problem — How c Fades Over Steps

The context vector c is injected only once — as the initial hidden state s₀. After that, it influences future decoding steps only indirectly, through the chain of hidden states. At each step it is passed through one more tanh transformation, progressively losing its signal.

Decoding step Output word Distance of c from this step Signal strength
Step 1 — s₁ "Le" c is directly present as s₀ Full ✓
Step 2 — s₂ "chat" c is 1 step removed (through s₁) Slightly diluted
Step 3 — s₃ "était" c is 2 steps removed (through s₁→s₂) More diluted
Step 10 — s₁₀ word 10 c is 9 steps removed Severely diluted ⚠
Step 30 — s₃₀ word 30 c is 29 steps removed Effectively lost ✗

For our short example "The cat sat" → "Le chat était", this works fine — there are only 3 decoding steps. But for a real sentence like "The prime minister announced a new economic policy yesterday afternoon" (10 words → perhaps 12 French words), by the time the Decoder reaches word 10 it has almost no reliable signal from the original English source. Early source words — which might carry the most important meaning — are effectively forgotten.

The context vector bottleneck — summarised
  • c is a single fixed-size vector regardless of input length
  • c is injected only once at the start of decoding
  • c becomes progressively less accessible as decoding continues
  • For long sentences, the Decoder loses track of the source partway through

7 Summary — The Three Things Each Step Relies On

# Step 1 — generating "Le"
s₁ = tanh( W·c          # context vector — DIRECT, full signal
         + U·emb(<s>)   # start token embedding
         + b_h )
"Le" = argmax softmax(V·s₁)

# Step 2 — generating "chat"
s₂ = tanh( W·s₁         # c is ONE step removed — slightly diluted
         + U·emb("Le")   # previous prediction fed back
         + b_h )
"chat" = argmax softmax(V·s₂)

# Step 3 — generating "était"
s₃ = tanh( W·s₂         # c is TWO steps removed — more diluted
         + U·emb("chat") # previous prediction fed back
         + b_h )
"était" = argmax softmax(V·s₃)

# Step N — for long sentences
sₙ = tanh( W·sₙ₋₁       # c is N-1 steps removed — severely diluted
         + U·emb(ŷₙ₋₁)   # previous prediction
         + b_h )
ŷₙ  = argmax softmax(V·sₙ)
Three things every decoding step relies on
  1. The previous hidden state sₜ₋₁ via W — carries memory of c and all prior predictions, but diluted with each step.
  2. The previous predicted word ŷₜ₋₁ via U — the model uses its own outputs as the next inputs (autoregressive generation).
  3. Learned weight matrices W, U, V — the same matrices are reused at every decoding step (parameter sharing).

What is missing: direct access to c at step 2, 3, 4 … — and direct access to the Encoder hidden states h₁, h₂, h₃. The Decoder cannot re-read the source sentence. It only has its running memory state.
This is exactly the problem Bahdanau Attention (2015) solved.

Instead of injecting c only once at step 1, Bahdanau's key insight was: let the Decoder re-read all the Encoder hidden states h₁, h₂, h₃ at every single decoding step — computing a fresh weighted sum each time.

When generating "chat" at step 2, instead of relying on a diluted memory of c through s₁, the Decoder directly attends to h₁ ("The"), h₂ ("cat"), h₃ ("sat") and decides which ones matter most right now.

Without Bahdanau:  s₂ depends on c only through s₁ — one level of indirection, diluted.
With Bahdanau:    s₂ attends directly to h₁, h₂, h₃ at this very step — fresh, direct access.
